# Contribuire al Conclave

Prima di tutto, grazie per l'interesse. Questo è un progetto di studio con supporto discontinuo, ma le contribuzioni sono benvenute.

## Come contribuire

### Segnalare un bug

Apri una [issue](../../issues) con:
- Cosa ti aspettavi che succedesse
- Cosa è successo invece
- La domanda che hai posto al Conclave (se possibile)
- Il contesto: quale versione di Claude stavi usando

### Proporre una modifica

1. Apri una issue per discutere l'idea prima di scrivere codice
2. Fai un fork del repository
3. Crea un branch descrittivo (`migliora-framework-filosofo`, non `fix`)
4. Fai le modifiche
5. Aggiorna il CHANGELOG.md
6. Apri una pull request con una descrizione chiara di cosa hai cambiato e perché

### Cosa è particolarmente utile

- **Test con domande diverse** — Hai provato il Conclave con un tipo di decisione interessante? Condividi cosa ha funzionato e cosa no.
- **Miglioramenti ai framework** — Se hai competenza in un framework analitico specifico (pre-mortem, 5 whys, ecc.) e vedi modi per rafforzare i prompt, è prezioso.
- **Bug report dettagliati** — Più contesto dai, più è facile capire e risolvere.
- **Traduzioni dei prompt** — Se vuoi testare prompt in altre lingue, condividi i risultati.

### Cosa evitare

- Pull request enormi senza discussione preventiva
- Modifiche che cambiano la struttura fondamentale senza aver aperto una issue
- Aggiungere dipendenze esterne allo skill

## Stile

- La documentazione del progetto è in italiano
- I prompt interni degli agenti sono in inglese (per efficienza del modello)
- I commit message possono essere in italiano o inglese, a tua preferenza
- Mantieni il tono informale ma chiaro

## Tempi di risposta

Questo è un progetto personale di studio. Potrei rispondere in un giorno o in un mese. Non è mancanza di rispetto — è il ritmo del progetto. Se la tua PR resta aperta a lungo, non significa che non sia apprezzata.
